# 版本 2.0：宋代历史人物 + 真实喝酒惩罚

## 吹牛酒肆 · 汴京酒楼版

**游戏名称**：吹牛酒肆 · 汴京酒楼版  
**主题**：北宋东京汴梁某家正店二楼雅间（完全按照《清明上河图》《东京梦华录》《营造法式》复原）。玩家是4位历史名人酒客，在珠帘绣额、藻井灯笼、窗外夜市灯火中吹牛、喝酒、比拼心机。

---

## 全新喝酒惩罚机制（v2.0）

替换原有简单命数扣减系统：

- 每位玩家面前固定摆放 **6瓶** 外观完全一模一样的酒坛（宋代典型瓷瓶/陶瓶）。
- 每位玩家自己的6瓶酒中**只有1瓶是蒙汗药**（位置在服务端随机固定，客户端不可见）。
- 每轮输家必须执行"选酒"动作：从自己面前剩余的酒坛中**手动选择1瓶**（点击或拖拽）。
- 选择后触发**真实喝酒骨骼动画**（角色举坛、仰头、喉结滚动、酒液下咽）。
- 动画结束后服务端结算：
  - 如果是安全酒 → 该酒坛消失（剩余瓶数-1），继续游戏；
  - 如果是蒙汗药 → 角色立即"醉倒"骨骼动画（身体摇晃→趴桌），该玩家淘汰。
- **最后一瓶永远是蒙汗药**（即使前面都没中，也会中）。
- 四个玩家各自独立6瓶酒，互不共享，可视化显示在每人座位前（酒坛随喝随少，视觉反馈极强）。

## 视觉要求（v2.0）

### 主场景
- 真实宋代二楼雅间（朱红八仙桌、高足椅、雕花屏风、珠帘、藻井、窗外汴京夜市）。
- 烛光动态摇曳效果（CSS keyframe / Canvas）。
- 窗外夜市光晕（暖黄/朱红渐变光源）。

### 酒坛系统
- 每人座位前清晰显示6瓶酒坛（初始满6瓶，喝一瓶少一瓶）。
- 酒坛外观一致（瓷白/米黄，宋代风格）。
- 被选中的酒坛高亮（金色描边 + 微微上浮）。
- 喝完后酒坛消失动画（缩小+透明）。
- 中毒时酒坛碎裂粒子效果。

### 角色动画（Spine骨骼）
每个模型至少包含以下骨骼动画状态：

| 动画名 | 触发时机 |
|--------|---------|
| `idle` | 默认站/坐姿，轻微呼吸浮动 |
| `shakeDice` | 摇骰子阶段 |
| `playCard` | 出牌时手臂前伸 |
| `selectBottle` | 手臂伸向选中的酒坛 |
| `drink` | 举坛仰头喝酒（含喉结动画）|
| `fallDrunk` | 中毒醉倒（摇晃→趴桌）|
| `slamTable` | 质疑时拍桌 |
| `laugh` | 对手中毒时大笑 |
| `quote` | 开场名言发言（摇头/举杯）|

## 技术实现概要（v2.0）

### Backend 变更
- `types.ts`：`Player` 新增 `characterId: string`（历史人物ID）、`characterModel` （骨骼模型类型 A/B/C/D）。
- `types.ts`：新增 `HistoricalCharacter` 接口、`CHARACTERS` 常量表。
- `rooms.ts`：`joinRoom` 支持 `characterId` 参数，写入玩家数据并推送给所有客户端。
- `ServerToClientEvents`：新增 `game:openingQuotes` 事件，游戏开始时推送所有玩家开场名言序列。

### Frontend 变更
- `CharacterSelectPanel.vue`：角色选择面板（分类浏览 + 搜索 + 预览名言）。
- `BottleRow.vue`：酒坛排列组件（6瓶 → 动态增减，点击高亮，消失动画）。
- `OpeningQuoteOverlay.vue`：开场名言顺序播放覆盖层（4角色依次弹出气泡）。
- `GameSeat.vue`：显示角色名、所用骨骼模型标识、剩余酒坛数。
- `Table.vue`：集成开场名言覆盖层 + 各座位酒坛行。
- `Lobby.vue`：加入游戏前显示角色选择面板。
- `gameStore.ts`：新增 `selectedCharacter`、`openingQuotes`、`showingOpeningQuotes` 状态。

# 原始设计文档（v1.0）

开发文档：吹牛酒肆 —— 一款4人联机吹牛 bluff 派对小游戏

## 1. 游戏概述

**创意名称**：吹牛酒肆  
**主题**：宋代江湖酒肆（古风水墨+赛博酒馆混搭）。  
核心卖点：100%还原"吹牛"文化，古风沉浸感，短平快（5-10分钟），适合微信群约战。

**玩家规模**：固定4人联机（支持房主创建私密房间+分享链接）。  
**胜利条件**：活到最后（每人2条命，输两次淘汰）。  
**平台**：纯Web（HTML5），后续用Taro/uni-app一键转微信小程序。

## 2. 游戏机制

### 2.1 骰子模式
- 每人1个酒盅+5颗骰子（服务端摇），逆时针喊话，质疑开盅结算。
- 惩罚：输家喝蒙汗药（减命）。

### 2.2 扑克模式（v2.0升级为酒坛惩罚）
- 20张酒令牌（6花雕/6女儿红/6竹叶青/2赖子），每人5张。
- 出牌+质疑，输家从个人6瓶酒坛中选一瓶喝（详见v2.0机制）。

## 3. 技术选型

- **前端**：Vue3 + Vite + PixiJS（2D渲染）+ Spine（骨骼动画）
- **实时**：Socket.io（Node.js）
- **部署**：Vercel / Railway
